GitHub Commit with Website Deployment

  1. Development Environment Setup Check - This is mostly here for me....
  2. Open CLI / New Terminal
  3. git --version
  4. node --version
  5. npm --version
  6. echo $path

  7. LET'S GET STARTED!

  8. Open CLI / Terminal

  9. In the Terminal / CLI

  10. Navigate into proper directory
  11. Create new folder - Type: mkdir nameHere
  12. cd into my newly created directory

  13. NOW TO GITHUB - Goto GitHub (sign in)

  14. Click on Plus Sign dropdown (upper right) and select Create New Repository
  15. Fill out Repository Name
  16. Fill out Description
  17. Public (radio button) is fine
  18. Select checkbox in Initialize with a README file.
  19. Add a README file
  20. Click green button - "Create Repository"
  21. Click green "Code" button - "Create Repository"
  22. Copy the https site url

  23. BACK TO CLI / TERMINAL

  24. Make sure I'm in proper folder/directory
  25. TYPE: git clone PasteTheHttpsIcopiedFromGitHub
  26. TYPE: ls
  27. See newly created GitHub repository
  28. TYPE: cd intoNewlyCreatedGithubRepositoryName
  29. TYPE: ls
  30. See README file
  31. TYPE: git status
  32. See "nothing to commit, working tree clean
  33. FINDER - I duplicated my working-title html file. I renamed my copied/duplicated working title html file to "index.html". I MOVED MY working title/new index.html html file - which I renamed to INDEX.HTML - FILE INTO MY NEWLY CREATED GITHUB REPOSITORY FOLDER - ALL IN FINDER WINDOWS!!!!
  34. Create index.html / site - - INSTEAD OF THIS - I already had my html/site/file created….I then clicked and dragged from original folder into newly created GitHub repository folder - - then I tipped git status and then the red untracked file came up.
  35. TYPE: git add .
  36. TYPE: git status
  37. See new file:
  38. TYPE: git commit -m "Info about my page."
  39. TYPE: git status
  40. See "Your branch is ahead of 'origin/main' by 1 commit.....use "git push" - nothing to commit, working tree clean
  41. TYPE: git push

  42. BACK TO GITHUB

  43. Refresh the GitHub webpage
  44. See newly created folder/files - Hooray!

  45. PREPARE THIS REPOSITORY/WEBPAGE FOR HOSTING THROUGH GITHUB

  46. Double-check I'm in the proper folder - containing my index.html file
  47. Click on Settings
  48. Click on Pages (left side of screen for me)
  49. Source: Deploy from a branch
  50. Branch: Main / root / CLICK ON SAVE BUTTON
  51. Wait for ten minutes - then refresh the web browser
  52. See your new site!
  53. Click Visit Site
  54. NOTEI choose to drag and drop. I did not move JOJ master folder over, rather, open the folder on my desktop, select and drag all the folders and files from within my Master Folder and drag them into the AWS upload box. Ex. go to my JOJ master folder, open it and select all the individual files and folder inside the joj master folder and drag all the individual folders and files into S3 Upload area.
  55. Click the Upload button
  56. After upload completes - get green Upload Succeeded!
  57. Single click on index.html file - Now taken to a new tab - will see owner, last modified, size, key. Can also - if lots of files and folders to fit on one page - maybe this is better: under “Files and folders” click on the file under the Name tab - - in search field “Find by name” - can input either index or the page I'd like to see.
  58. NOTE: the OBJECT URL (inside the properties tag in the Object Overview box) - this represents the file inside AWS - and can be used for public access - but, just because AWS creates this file usable for public access does not mean that the html file / this link is accessible
  59. TRY opening the Object URL link in another tab - will get error - XML file does not appear to have any style info. Code Access Denied, Message Access Denied, etc. This means that the file is not made public (yet) - I don not have the proper public permissions yet
  60. Go back to Object Overview tab - In the same index.html overview tab, click the Open button. Now I can see the file contents. NOTE: the website won’t show up as finished product - only the html shows up - the skeleton - latest-my photos didn't show up. Example, click any link and I get the XML error / Access Denied message. Simply uploading all of our files does not mean our website is going to work. We need to complete more steps. In the above steps, I created my bucket and uploaded my files.

  61. Next Part - configure my bucket to host my static website.

  62. Navigate to AWS S3 - Top Left: Click Services (make sure I'm still logged in)
  63. Click S3
  64. Click on my bucket
  65. Select PROPERTIES tab
  66. Scroll down - In STATIC WEBSITE HOSTING area Click Edit button - This is the configuration I need to set to enable my bucket to host my website
  67. Under “Static website hosting” Select ENABLE (Radio Button) / more info appears
  68. Select - Enable - under static website hosting
  69. More options present themselves — go to index document text box and type in index.html - to denote that this is the document I want displayed when someone types in my url - this is my main homepage
  70. Optional - enter in the error.html document - the document that gets displayed when a user tries to access pages that are not available - can make this page really cool - either way, enter an html document here - with directions and a link back to the homepage. Note - this is the page with the redirection rules and notes. Optional, but considerate. It's good form.
  71. Click Save Changes button - I got a green note and check mark - “Successfully edited static website hosting” message in green bar up top
  72. Doublecheck that the static website hosting is enabled - scroll down on the properties tab/page - it should have some info in it now - remeber to copy the http link created for my site
  73. Above, we generated an endpoint URL - and configured my error page NOTE: my endpoint url - it looks something like this: http://MY-BUCKET-NAME.s3-website-us-east-1.amazonaws.com Note - when I tried to open this, I got the 403 Forbidden message. Access denied - - This is normal, see below…

  74. Next Part - configure security and Permissions - to make my site public for users on the internet

  75. Navigate back to AWS S3 - Top Left: Click Services (make sure I'm still logged in)
  76. Click S3
  77. Click on my bucket
  78. NOTE: by default s3 objects are not made public even when bucket hosting is enabled. Even if an endpoint is configured, still no go!!! - trying it gives us an hsttp 403 error - access denied - go ahead - paste my endpoint url and will get 403 forbidden. To remedy this, need to perform 2 actions in my bucket: 1 - Need to unblock public access to my bucket, AND 2 - make my website files public
  79. Goto AWS console - click services - Storage-S3
  80. Click on the bucket I want to open configuration tabs
  81. Click on PERMISSIONS TAB
  82. Click on EDIT under Block public access (bucket settings)
  83. UNCHECK box “Block all public access”
  84. Doublecheck that all other checkboxes in this section are unchecked
  85. Click SAVE CHANGES button - Caution box appears
  86. Type in “confirm” in the caution box
  87. Click Confirm Button
  88. Click on Objects tab
  89. Click the REFRESH ICON - the circle with the arrow pointer right/clockwise

  90. DO THE EDIT BUCKET POLICY NOW

  91. Select Permissions tab
  92. Click Edit Button in Bucket policy section
  93. Paste THIS code in the text field under “Policy:”
  94. {
                              "Version":"2012-10-17",
                              "Statement":[
                                {
                                  "Sid":"PublicReadGetObject",
                                  "Effect":"Allow",
                                  "Principal": "*",
                                  "Action":["s3:GetObject"],
                                  "Resource":["arn:aws:s3:::your-bucket-name/*"]
                                }
                              ]
                            }
  95. Above - where it says “your-bucket-name” = put in your bucket name!!!!! And Keep the /*
  96. Click Save changes button
  97. OK - go try the site
  98. Services > S3 > Select Bucket > Click refresh button > Properties tab > scroll to Static website hosting > click the link - opens in a new tab
  99. Check your work!
  100. Start debugging!
